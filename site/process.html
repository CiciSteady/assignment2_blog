Process

How the website was built

The website was created as a no-dependency static project so it can run directly inside WSL and remain easy to deploy with GitHub Pages or a local IP-based server.

01

Initialize project structure

Create a dedicated repository, add folders for pages, assets, downloads, docs, and scripts.

mkdir -p assignment2_blog/site/assets assignment2_blog/site/downloads assignment2_blog/docs assignment2_blog/scripts
02

Build the landing page

Design a responsive homepage with navigation, summary cards, and a visible quick-start command block.

git commit -m "feat: build responsive landing page and styles"
03

Integrate Assignment 1 materials

Copy the previous report into the static site and expose it through download buttons.

cp ../assignment1_report.md site/downloads/
cp ../assignment1_report.pdf site/downloads/
04

Document the full process

Write a Markdown report and a copyable command guide that explain the technical decisions and workflow.

git commit -m "docs: add assignment2 report and WSL command guide"
05

Prepare deployment

Add a WSL serving script, an IP helper script, and a GitHub Pages workflow for an optional public URL.

python3 -m http.server 8000 --directory site --bind 0.0.0.0

Deployment

Local URL or IP address

Start the local server from WSL and access the website through a browser:

cd /mnt/c/Users/r1382/Desktop/codex/assignment2_blog
bash scripts/get_wsl_ip.sh
bash scripts/serve.sh 8000

Then open either http://localhost:8000 or http://YOUR_WSL_IP:8000.

Optional public deployment

The repository also includes a GitHub Pages workflow so the site can be published as a public URL after pushing the repository to GitHub.

All pages are static and require no build step.
